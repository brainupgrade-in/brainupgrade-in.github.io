What AgentGrow Does

Your AI Assistant for Everyday Browser Tasks

AgentGrow reads and interacts with any webpage in real-time, helping you automate the tasks that eat your time — click buttons, navigate pages, fill forms, write emails, summarize documents, and more.

📝

Browser Task Automation

Fill forms, click buttons, navigate pages, write drafts, compose messages in any web app. Tell AgentGrow what to do in plain English and watch it interact with the page for you.

👆

Click & Navigate

Click buttons, follow links, submit forms — all from chat. AgentGrow scrolls to the element and clicks it with React/Angular/Vue-compatible events. Browse the web without lifting your hands from the keyboard.

🛡

Action Safety Mode

"Ask before acting" (default) shows what AgentGrow will do and waits for your approval. Switch to "Auto-act" for power users — with a persistent risk warning. You stay in control.

🎯

Smart Selector Resolution

5-level fallback (ID, name, aria-label, placeholder, positional) ensures form filling works across Google, Amazon, AWS, Salesforce, and any SPA. No brittle CSS selectors — it finds the right element every time.

🌐

Any LLM Provider

7 built-in presets: OpenRouter (300+ models), OpenAI, Anthropic, Groq, Google Gemini, Ollama on localhost, plus Custom for any OpenAI-compatible endpoint. No vendor lock-in.

🏠

Private Network Support

Works with local and on-premise LLM servers (Ollama, vLLM, LM Studio) on 192.168.*, 10.*, 172.16.* without CORS issues. Your data never leaves the corporate network.

🔍

Live Page Intelligence

Auto-reads the adjacent tab content in real-time — headings, forms, code blocks, links, selected text. No screenshots, no snapshots. Live, structured DOM access.

🧠

Smart Context

Auto-detects text selection vs full page. Always includes form fields with CSS selectors so the AI can target them precisely. No manual toggles needed — it just works.

🔄

One-Click Model Switching

Switch between models directly from the chat dropdown. All preset models for your active provider are available instantly — no digging through settings pages.

📋

Form Filling & Writing

Fill any form field including React, Angular, and Vue apps (dispatches native events). Write to contenteditable areas — Gmail compose, Telegram, Slack, Notion, and more.

🔒

Privacy-First

API keys encrypted with AES-GCM-256. Zero data sent to AgentGrow servers — there are none. Full source code on GitHub (MIT). You own and control everything.

💬

Chat Persistence

Conversations are saved locally and restored when you reopen the side panel. Pick up exactly where you left off — no lost context, no re-explaining.

⚠️

Smart Errors

Specific, actionable error messages: invalid API key, rate limited, model not found, network timeout. Every failure includes a retry button — no guessing what went wrong.

Verifiable by Design

Built to Be Trusted

Every trust claim in AgentGrow is verifiable — not just stated. Here's how we make privacy and transparency something you can actually check.

Open Source

Read Every Line

The full source is on GitHub under MIT license. No obfuscation. No minified-only builds. You can read every line, fork it, and build from source yourself.

Encrypted Keys

AES-GCM-256 At Rest

API keys are encrypted using the Web Crypto API before being written to storage. They are never readable in plaintext, never sent to content scripts, never exported.

HTTPS Enforcement

Remote Calls Are Always Encrypted

The extension rejects HTTP provider URLs for any non-localhost endpoint. Local Ollama and LM Studio on private networks are explicitly allowed.

No Telemetry

Zero Analytics, Zero Tracking

AgentGrow has no analytics SDK, no crash reporter, no usage metrics. The only network calls it makes are to your configured LLM endpoint and Google Fonts for typography.

Session-Only Auth

Sign-In Cleared on Close

Your Google OAuth token is stored in session storage only — cleared automatically when you close Chrome. Sign-out immediately revokes the token and clears all session data.

Strict CSP

No Remote Code Execution

The Content Security Policy blocks eval(), inline scripts, and remote script loading. All JavaScript is bundled at build time. Markdown output is sanitized with rehype-sanitize.

Privacy Policy

Effective Date: April 11, 2026 Last Updated: April 11, 2026 Extension: AgentGrow — AI Browser Extension Publisher: Gheware Technologies

📋 Short version: AgentGrow stores your data only on your device, encrypted. It sends your messages only to the LLM endpoint you configure. It collects zero analytics, zero telemetry, and has no servers of its own.

1. Who We Are

AgentGrow is developed and maintained by Gheware Technologies ("we", "our", "us"), a technology training and tooling company based in India. Contact: contact@gheware.com

2. Information AgentGrow Collects

AgentGrow collects and stores the following data locally on your device only:

  • Conversation history: The messages you send and receive within AgentGrow. Stored in chrome.storage.local, which is accessible only to the AgentGrow extension on your device.
  • Provider configurations: The API endpoint URL, provider name, and selected model you configure. Your API key is stored encrypted (AES-GCM-256) and is never readable in plaintext after entry.
  • Prompt templates: Any templates you save for reuse.
  • User preferences: Theme, default context mode, and keyboard shortcuts. Non-sensitive preferences are synced via your Google account's chrome.storage.sync.

3. Information AgentGrow Does NOT Collect

AgentGrow has no servers, no backend, and no analytics infrastructure. The following data is never collected:

  • Your browsing history or the URLs you visit
  • The content of your conversations (beyond local storage on your device)
  • Personal identifiable information of any kind
  • Crash reports or error logs sent to external services
  • Usage analytics, feature tracking, or session data
  • Your IP address or device fingerprint

4. How Information Is Used

Data stored locally is used solely to:

  • Display your conversation history within the extension
  • Authenticate requests to your configured LLM provider
  • Remember your UI preferences

5. Data Sharing

AgentGrow shares data with one party only: the LLM API endpoint you configure in Settings. When you send a message, that message (and optionally the current page text, if you have enabled context mode) is sent to your configured endpoint.

⚠️ Important: The privacy practices of your chosen LLM provider (OpenRouter, OpenAI, Groq, your company's internal server, etc.) are governed by that provider's own privacy policy. AgentGrow has no control over, and no visibility into, what that provider does with the data you send. Please review your provider's policy before use.

We do not sell, share, rent, or disclose your data to any other third party.

6. Data Security

  • API key encryption: API keys are encrypted using AES-GCM-256 via the Web Crypto API before storage. They cannot be decrypted without the per-device encryption key, which is also stored locally.
  • HTTPS enforcement: AgentGrow refuses to send API calls over HTTP to non-localhost endpoints.
  • Content Security Policy: The extension enforces a strict CSP that prevents execution of any remote code.
  • Isolated architecture: API keys and provider configurations are accessible only in the extension's service worker, never in content scripts or web pages.

7. Data Retention and Deletion

Your data is retained locally until you delete it. You can delete provider configurations within the extension settings. To remove all data, uninstall the extension — Chrome automatically deletes all data stored in chrome.storage.local.

If you uninstall the extension, all data stored in chrome.storage.local is deleted by Chrome automatically. Sync storage is removed when you disconnect your Google account from Chrome Sync.

8. Children's Privacy

AgentGrow is not directed to children under 13. We do not knowingly collect information from children. If you are under 13, please do not use this extension.

9. Your Rights (GDPR / CCPA)

Because all data is stored locally on your device and we have no servers, we cannot access your data to fulfil erasure or portability requests through our systems. However:

  • Access: All data is stored locally on your device in Chrome's extension storage.
  • Portability: Data export functionality is planned for a future release.
  • Erasure: Uninstall the extension to delete all stored data. Chrome automatically removes all extension storage on uninstall.
  • Correction: Edit provider configurations directly within the extension settings.

If you have questions about your rights, contact us at privacy@gheware.com.

10. Changes to This Policy

We will post any changes to this Privacy Policy on this page and update the "Last Updated" date. For material changes, we will provide a notification within the extension. Continued use of AgentGrow after changes constitutes acceptance of the updated policy.

11. Contact

For privacy-related questions or concerns:
